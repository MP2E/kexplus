Level Editing
-----------------------------------------------------------------------

* ---------Introduction---------

This article will assume that you're already familiar with Doom editing. This article will also use a lot of technical descriptions such as linedef, sector, subsectors, mobj etc. For more information on the Doom engine, check out http://doom.wikia.com/wiki/Entryway


    * Level Contruction Workflow
    Its really ideal to do a rough outline of your level in the editor. That way you can get a
    clear idea of the flow, and length of the level. A rough outline is basically contructing
    the level without focusing on texturing, detail, or sector heights. In Doom64, its also ideal
    to wait until the level is in a fully playable state and detailed before working on the
    lighting.

    This method is not necessary but it can speed up the workflow and prevent having to frequently
    redo parts of the level.

    * Devparm

    Doom64 re-implemented the -devparm feature which is extremly beneficial to the mapper.
    Devparm offers features for debugging gameplay, levels, and lighting.

    When enabling devparm, the following features are enabled:

        * Demo Mode: Press F1 to turn this mode on. This will allow you to take damage, but never
          die (health stays above 10). This is extremly useful to determining how difficult your
          level is without knowing when using regular god mode cheats.
        * Spectator Mode: Press F2 to turn this mode on. The player will be able to quickly fly
          and clip throughout the entire level.
        * Lock Monsters Mode: Press F3 to turn this mode on. This will freeze all monsters in
          the level. Useful for debugging a script or taking screenshots.
        * Kill all monsters: Pressing F4 will kill all monsters in the level.
        * Wireframe Mode: Pressing F7 will toggle wireframe mode.
        * No Lights: Pressing F8 will turn off all lighting.


    * Setting up Doom Builder (aka Builder64)

        * Unzip Builder64 into directory of choice
        * Edit Doom64.cfg and define the path for "texturedirectory" in order for the editor to
          load up the .png textures.
          * Example: texturedirectory = "E:\\sources\\Doom Builder\\Dev\\Textures";
          * Make sure to use double backslashes instead of using one backslash.
        * For the IWAD, set "Builder.wad" as the main iwad which only contains the playpal data in
          order to properly view the textures.
          * Sprite support is not yet implemented due to the large number of sprites to incorporate 
            into the editor.
        * Please use the included glbsp executable for node building only which has been customized
          to support the doom64 map format.


    * Changes from regular Doom Builder

        * Colors for invisible lines and macros, etc.
        * Default color changed
        * Texture directory parameter
        * Slightly improved sector selection
        * Textures no longer read from texture/pnames lump as they're no longer needed
        * D64 format supported (other formats no longer supported).
        * Thing dialog window simplified

    * Extracting Doom64's levels
        * Open a lump editing program such as XWE or Slumped
        * Open DOOM64.WAD
        * Extract lumps Map01 through Map33.
        * Rename the extention from .lmp to .wad

    * Known Issues

        * 3D mode not working due to using GLSEGS as default level segs.
        * Handling lighting for multiple selected sectors needs improvment
        * Texture/sprite/internal palette handling needs improvment
        * Bug where after compiling level with node builder, user must save his level to save the
          compiled lumps into the wad.
        * Builder will not build the nodes correctly until level contains at least 100 linedefs
          and at least 20 sectors.


* ---------Level Basics---------

    * Compiling

        * This version of GLBSP.exe no longer support the -normal option nor V2-V5 vertices. Simply
        use the option glbsp.exe <input wad> -o <output wad>.
        * For Doom Builder, everything should already be configured. Compiling the level in the
          editor should be the same as compiling in the original Doom Builder.
        * After compiling in the editor, make sure the level is saved again.

    * Testing

        * Testing a custom level can be done through the classic way by using -file or by launching
          the level from the editor.

    * Using the /MAPS Directory

        * Doom64 EX can be able to dynamically load custom wads in the /maps folder. This folder
          should be located in the main Doom64EX directory (folder containing the
          Doom64 executable).
        * To load the custom wad, go the main menu and select Custom Levels.
          The custom wad can now be selected in the menu.
        * As of now, only single level wads are supported.
