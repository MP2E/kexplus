Linedef Specifications
-----------------------------------------------------------------------
Doom64's linedefs are completly different than that of Doom, Zdoom/Hexen's format. This
Article will roughly go over the new format and all of its flags and specials.

* ---------Linedef Format---------

    * short v1;
    * short v2;
    * unsigned short flags;
    * byte renderflags;
    * byte sidedefflags;
    * byte special;
    * byte usetype;
    * short tag;
    * short sidenum[2];


* ---------Linedef Flags---------
    * Normal Flags
        * Impassible
        * Block Monsters
        * Two Sided
        * Unpeg Top
        * Unpeg Lower
        * Secret
        * Block Sound
        * Invisible
        * AutoMapped
        * Mask Texture - For any texture with masking (fence, grate etc) this flag should be set.
        * Trigger By Event - Linedef will be triggered by either thing pickups, or death.
                             Linedef's tag must match the thing's TID. If tag is 999, then the
                             linedef will trigger once the level loads.
        * Switch Mask 1 - Read below
        * Switch Mask 2 - Read below
        * Switch Mask 3 - Read below

        * Switch Masks
          
          Switch masks are special flags that control where the switch line should be displayed
          and on what sidedef slot (top, middle, low) it should be activated from. This means
          that if a switch is activated on low sidedef, then the player will not be able to
          use the switch if he is not able to 'reach' it.

          A switch line is a generated 32x32 plane that is used to display the switch. One of the
          three switch mask flags sets how the switch is displayed.

          A switch texture is any one of the textures named SWXxxxxx. These textures must be placed
          in either top, middle or lower sidedef texture slot. All switches must be used on
          the front side of a linedef.

          Flag                 Sidedef Texture        Sidedef Activation
          --------------------------------------------------------------
          Mask1                Top                    Low
          Mask2                Middle???              Top???
          Mask3                Low???                 Middle???
          Mask1+Mask3          Top                    Middle
          Mask1+Mask2          Middle                 Low
          Mask2+Mask3          Low                    Top

          ??? - Not confirmed. Doesn't appear to be even functional in the rom.


    * Usage Flags

        Usage flags are flags based on how the linedef can be activated. This can range from
        crossing, pressing use, shooting, key-based etc.

        * Macro      - If this linedef is triggered, it will trigger a Macro script
        * Red Key    - Requires red key to trigger the line
        * Blue Key   - Requires blue key to trigger the line
        * Yellow Key - Requires yellow key to trigger the line
        * Cross      - Player must walk over, or cross the line to trigger it
        * Shootable  - Player must shoot at the line to trigger it
        * Use        - Player must press use on the line to trigger it
        * Repeatable - Line can be reactivated

    * Sidedef Flags

        Sidedef flags are flags based on sidedef/texture behavior.

        * Trigger Front Side     - Line can only be triggered on the front side of the line
        * No Check for Z Height  - Line won't check for the player's Z height (by default it
                                   checks the player's Z height and the coordinates of each sidedef
                                   texture height (top, middle, lower).
        * Invert Blend           - Reverse color blending (Example: Red pillars in Dark Entries).
        * Mirror Texture H       - Mirror textures horrizontally
        * Mirror Texture V       - Mirror textures vertically

    * Rendering Flags

        Rendering flags are flags based on how they're rendered (lighting, animation etc).

        * Switch           - Unknown. Used on some switches, but not on (Mask2+Mask3) switches.
        * Scroll Right     - Scroll texture to the right
        * Scroll Left      - Scroll texture to the left
        * Scroll Up        - Scroll texture upwards
        * Scroll Down      - Scroll texture downwards
        * Limit Top Blend  - Blend Top/Low color on top sidedef only
        * Limit Low Blend  - Blend Top/Low color on low sidedef only
        * Sector Blending  - Use standard color values for blending. (If not on, it will use thing
                             color on the entire sector without blending (psxdoom-ish lighting).


        Below are examples of how the blending works:

        Example 1: Linedefs with sector blending enabled. The top and bottom seg colors are blended
                   together. (Ceiling = Green, Floor = Magneta, Upper wall= Red, Lower wall = Blue)
                  
        Example 2: Linedefs without sector blending enabled. The linedef instead uses the thing
                   color (cyan) to light the wall.
                    
        Example 3: Linedef without Limit Top and Limit Bottom blending. The blending between the
                   two colors are spreaded out between the top and bottom sidedefs (middle sidedef
                   remains uneffected).
                    
        Example 4: Linedef with both Top and Bottom Limit blending. The top and bottom colors are
                   being blended on each sidedef (including middle).
                  
        Example 5: Linedef with Top Limit blending and No Bottom Limit Blending. Top Sidedef has
                   full blending between the two colors, while the bottom sidedef does not.
